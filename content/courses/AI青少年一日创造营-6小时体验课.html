AI 青少年一日创造营（6 小时体验课）

核心定位： 不是教 AI 工具，不是编程营，而是一场少年 AI 创新者的启蒙体验。
核心理念： 创新不是因为想出好主意，而是因为发现了真实的问题。每个孩子都可以成为用 AI 解决社会问题的人。

>

设计理念： 大道至简。一天体验课做下来，家长看见孩子的社会关怀，孩子看见自己改变世界的能力。

课程总览

| 项目 | 内容 |

|------|------|

| 时长 | 6 小时（1 天） |

| 设备 | 仅需手机（全程手机完成） |

| 适合年龄 | 8-14 岁 |

| 每班人数 | 12-20 人（分 4 组，每组 3-5 人） |

| 师资配置 | 1 主讲教练 + 2 助教 |

| 核心成果 | AI 项目海报 + AI 互动游戏 + 灵光作品集 + EdgeOne 个人网站 + 路演展示 |

上午：认识自己，确定项目（3 小时）

09:00-09:30 │ 报到破冰

目标： 消除陌生感，建立团队安全感

流程：

  • 签到拍照：给孩子拍一张"入营照"
  • 破冰游戏：颜色 + 情绪 + 名字
  • "我是红色的开心的杨杨"
  • 全员跟读鼓掌：红色！开心！杨杨！
  • 分组：4 人一组，选组长
  • 约定规则：
  • 音乐响起 30 秒内回到座位
  • 同学上台：鼓掌 + 微笑 + 竖大拇指
  • 大胆说，不怕错

教练要点：

  • 助教协助拍照，记录每个孩子的初状态
  • 观察谁主动、谁安静，记录在《学员观察表》

09:30-10:30 │ 戏剧教育：我是谁

目标： 让孩子打开自己，表达自己，看见自己

环节 1：身体热身（10 分钟）


「情绪雕塑」
老师说情绪词（开心/生气/害怕/勇敢）
孩子用身体摆出造型，不能说话
→ 打开身体，释放表达欲

环节 2：我是谁 — 角色扮演（20 分钟）

"假如你可以变成任何一个你想成为的人或物，你会是什么？"

每组选一个主题：

  • 🌊 海洋守护者（环保主题）
  • 🦸 校园英雄（反霸凌主题）
  • 🚀 未来探险家（科技主题）
  • 🌳 森林精灵（自然主题）

玩法：

  • 每个孩子用手机录一段 30 秒短视频自我介绍
  • "我是 XX，我是 XXXX（角色名），我的超能力是______"
  • 组内互看视频，投票选出组代表
  • 代表上台用角色身份完成 1 分钟即兴表演

教练要点：

  • ⚠️ 这一步至关重要—孩子的角色选择直接反映内心关注点
  • 选择"海洋"的孩子可能关心环境
  • 选择"校园"的孩子可能经历过或见过校园问题
  • 这些信息就是家长想看见的"孩子的心"

环节 3：AI 对谈（15 分钟）

用豆包 App 完成一次"我是谁"深度对话

在手机上操作：


打开豆包 App → 开始语音对话，说：

"我今年 X 岁，我的角色是________，
我想用 AI 做一件帮助________的事，
请采访我，每次问 3 个问题，
帮我找到我最想做的项目方向。"

成果： 每个孩子获得 3 个可能的项目方向

教练巡场：

  • 帮助打字慢的孩子使用语音输入
  • 引导孩子把 AI 的回答读出来

10:30-11:30 │ AI 问答做项目海报

目标： 从"想法"到"看得见的作品"

步骤 1：确定项目方向（10 分钟）

孩子从 AI 对话结果中选一个最心动的方向。

常见项目方向参考：

| 角色主题 | 项目方向 | 海报内容 |

|---------|---------|---------|

| 海洋守护者 | 减少塑料垃圾 | 我的海洋清洁计划 |

| 校园英雄 | 反霸凌 | 勇气守护行动 |

| 未来探险家 | AI 改变学习 | 未来学习法 |

| 森林精灵 | 保护动物 | 动物朋友在呼救 |

步骤 2：用豆包生成项目海报（30 分钟）

手机操作流程：


打开豆包 App → 打开"AI 生图"功能 → 输入 prompt：

"我是一个 X 岁的孩子，
我的项目名称是________，
我想解决________问题，
请帮我生成一张项目海报，
海报上要有：
1. 项目名称
2. 我的名字
3. 团队名称（如果有）
4. 一句 Slogan"

效果好的 prompt 示例：


我是 10 岁的杨杨，我的项目叫"海洋清道夫"，
我想解决海洋塑料污染问题，
请帮我生成一张海报，要有项目名称、我的名字、
团队名"蔚蓝战队"、Slogan"还给海洋一片蓝"。
海报风格要色彩鲜艳，适合青少年。

助教任务：

  • 确保每个孩子都成功生成了至少 1 张海报
  • 海报保存到手机相册

步骤 3：小组分享（20 分钟）

每组轮流展示海报，每个孩子 2 分钟：

  • 这是我的项目海报
  • 我想解决的问题是...
  • 我的 Slogan 是...

教练点评要点：

  • 首先肯定：每个创意都值得被看见
  • 追问一句：你觉得谁最需要这个项目？
  • 引导思考：你的项目能给多少人带来帮助？

11:30-12:00 │ 分享展示 + 教练点评

形式： 全班围坐，每组推选 1 人上台展示

教练总结话术：

"今天上午，你们每个人都找到了一个自己关心的问题。有人想保护海洋，有人想帮助同学，有人想发明未来的学习方式。这些关怀非常珍贵。下午，我们要用 AI 把这些关怀变成真实的方案。"

记录工具： 教练填写《学员观察表》


学员观察表示例：
姓名：______  组别：______
角色主题：________
项目方向：________
表达力：⭐ ⭐ ⭐ ⭐ ⭐
主动性：⭐ ⭐ ⭐ ⭐ ⭐
合作度：⭐ ⭐ ⭐ ⭐ ⭐
教练备注：________

🍱 12:00-13:30 午休

  • 播放轻音乐 / 眼保健操
  • 教练整理上午观察记录

下午：用 AI 创造（3 小时）

13:30-14:30 │ AI 创作游戏（灵光创作广场）

目标： 每个孩子用灵光创作第一个 AI 互动游戏

为什么用灵光？ 灵光的创作广场有大量现成的游戏和应用模板，孩子可以：

  • 看到别人做的作品，激发灵感
  • 直接修改模板，比自己从零写代码快得多
  • 一键发布分享，无须折腾部署

第一步：注册灵光 + 浏览创作广场（5 分钟）

手机操作：


1. 打开灵光 App → 手机号登录
2. 底部导航点"创作" → 进入「创作广场」
3. 浏览精选作品：游戏、问答、故事…
4. 搜索关键词：环保、反霸凌、海洋、未来…

教练大屏演示：

"看，这些都是跟你们年龄差不多的孩子用灵光做的作品。有环保闯关游戏，有反霸凌问答，有未来学校……今天下午，你也能做出一个。"

第二步：找一个模板/灵感（10 分钟）

孩子根据上午确定的项目方向，在创作广场中找到感兴趣的作品作为参考。

找不到完全匹配的？没关系，关键是思路。 比如做"海洋保护"的游戏，可以参考一个"垃圾分类"的模板，把主题换掉就行。

教练巡场引导：

  • "你喜欢这个？好，我们可以参考它的玩法，换成你的主题。"
  • "这个游戏跟你做的项目方向很像，对吧？"

第三步：用灵光创作游戏（25 分钟）

路径 A — 基于模板修改（推荐，15-20 分钟）：


1. 在创作广场找到想参考的作品
2. 点"创作同款"或"查看源码"
3. 修改内容：
   - 标题 → 改成你的项目名称
   - 题目/关卡 → 改成你的项目主题
   - 图片/文字 → 用上午生成的海报/概念
4. 点"预览"测试效果
5. 满意后点"发布"

路径 B — 从零用 AI 对话创建（15-25 分钟）：


1. 在灵光创作页，点"AI 创作"
2. 用语音或文字描述你的想法：

"帮我做一个关于海洋保护的问答闯关游戏，
有 5 道题，每道题选对得 20 分，
画面要蓝色调，适合青少年。"

3. 灵光会自动生成
4. 预览 → 根据 AI 建议调整 → 发布

路径 A 适合动手能力强的孩子，路径 B 适合更依赖引导的孩子。教练根据每个孩子的情况推荐路径。

第四步：分享游戏链接（10 分钟）

发布后，灵光会生成一个分享链接。 孩子把链接发到小组群。

小组互玩环节：

  • 点开同伴的游戏链接，试玩
  • 说一个你最喜欢的点
  • 给对方一个改进建议（一个就行）

教练引导：

"看到没有？从想法到一个能玩的游戏，你只用了不到 30 分钟。这就是 AI 时代创造的速度。"

出现问题时的应对：

  • 灵光生成不理想？换个 prompt 再试，豆包也可以辅助构思
  • 模板修改不成功？找助教帮忙，旁边有备用案例可以直接用
  • 关键还是那句话：让孩子体验"我做到了"

灵光用不上的备选方案

如果个别孩子手机装不了灵光 / 登录失败，用回豆包写 HTML，通过 EdgeOne 部署的方案：


豆包 → "帮我写一个XX主题的 HTML 小游戏"
→ 复制代码 → 发到教练电脑 → 助教上传至 COS
→ EdgeOne 加速 → 返回链接给孩子

14:30-15:30 │ 作品集 & 路演准备

目标： 把上午的海报 + 下午的游戏，汇聚成个人作品集 + 通过 EdgeOne 部署到线上

第一步：整理成果清单（10 分钟）

每个孩子准备以下素材：

  • ✅ 项目海报（上午豆包生成，存手机相册）
  • ✅ AI 游戏（灵光已发布，有分享链接）
  • ✅ 项目 Slogan（上午确定）
  • ✅ 自我介绍（我是谁 + 我想改变什么）

第二步：用灵光搭建个人作品集（15 分钟）

灵光的最大优势之一：海报和游戏可以在同一个平台上集中展示。

手机操作：

路径 A — 用灵光的"作品集"功能（推荐）：


1. 打开灵光 → 点右下角"我的" → 我的作品集
2. 新建作品集 → "我的项目展示"
3. 添加内容：
   - 标题：项目名称
   - 描述：项目 Slogan + 一句话介绍
   - 添加上午的海报图片（从相册上传）
   - 关联下午发布的游戏（从已发布作品中选择）
4. 调整顺序 → 保存 → 生成分享链接

完成标志： 每个孩子手机上都有一条自己的作品集链接，打开后能看到海报 + 游戏 + 项目介绍。

第三步（亮点）：用腾讯云 EdgeOne 部署个人网站（15 分钟）

这是全课最"哇"的时刻——每个孩子的项目变成一个真正的网站，通过 EdgeOne 加速分发。

为什么用 EdgeOne 而不是临时工具站？

  • 链接长期有效，课程结束后也能访问
  • EdgeOne 边缘加速，打开飞快
  • 用的是企业级基础设施，不是临时在线工具
  • 家长可以转发给亲戚朋友看，传播效果拉满

教练电脑端操作流程（大屏投屏演示）：


【课前已配置好】:
1. 腾讯云 COS 存储桶 → 已开启静态网站托管
2. EdgeOne 加速域名 → 已绑定 COS 源站
3. C polar 客户端 → 已预装在教练电脑

现场部署流程：


第 1 步：孩子用手机把"自我介绍 + Slogan + 游戏链接"发到教练微信

第 2 步：教练/助教在电脑上，用豆包网页版生成每个孩子的个人 HTML：

豆包 Prompt：
"帮我生成一个青少年的个人项目展示页，
姓名：XXX，年龄：X 岁，
项目名称：XXX，
Slogan：XXX，
游戏链接：XXX，
海报描述：XXX（占位图即可）
设计要色彩鲜艳、儿童友好、适合手机浏览。
所有代码放在一个 HTML 文件里。"

第 3 步：保存为 index.html → 打开 C polar → 
       选择 COS 存储桶 → 上传文件

第 4 步：C polar 点击"复制链接" → 发回孩子的微信

第 5 步：孩子在手机上打开链接 → EdgeOne 加速加载 → 
       "这是我的网站！"

教练话术：

"这个链接不是临时有效，它会一直在。你回家可以发给爸爸妈妈、爷爷奶奶、同学朋友看。这就是你在这个世界上留下的第一个网站。"

时间不够时的简化方案：

  • 助教提前用豆包生成好 3-5 个通用模板页面，现场只需替换名字和链接
  • 使用 COS 的批量上传功能，一次性部署所有页面

完成标志： 每个孩子微信里都收到一条 EdgeOne 加速链接，打开就是自己的项目网站。

第四步：路演准备（15 分钟）

每个孩子准备 2 分钟路演：

"大家好，我是______。
我的项目是______，因为我发现______问题。
这是我的海报（展示手机海报），这是我的游戏（打开灵光展示）。
我的 Slogan 是______。
谢谢大家！"

小组内先演练一遍。

教练巡场指导：

  • 帮孩子把路演词缩短到核心：问题 → 方案 → 成果
  • 鼓励孩子用自己的话，不要背稿
  • 内向的孩子可以只讲 3 句话，助教在旁边举着手机展示作品

15:30-16:00 │ 创造者发布会（路演）

氛围营造：

  • 播放入场音乐
  • 现场布置：拉一个"AI 创造者发布会"横幅
  • 邀请家长/教练团队当观众

路演流程：


主持人： "欢迎来到 AI 未来创造者发布会！今天，每个孩子都是创造者！"

第一组 上台（每人 2 分钟）
  → 展示海报（手机相册 / 灵光作品集）
  → 展示游戏（打开灵光，现场玩一下）
  → 展示 EdgeOne 个人网站（打开链接，这是我的网站！）
  → 说出创造宣言

教练点评（每组结束后 2 分钟）
  → 先肯定一个亮点
  → 再给一个建议

... 依次进行 ...

颁奖环节：
🏆 最佳社会关怀奖
🏆 最佳创新方案奖
🏆 最佳表达奖
🏆 最佳团队奖
🏆 未来创新者奖（每人都有）

点评原则： 每个孩子必须至少获得一个正向反馈。没有"最差"，只有"不同的优秀"。

教练话术参考：

  • "你的海报色彩选择非常有冲击力！"
  • "你的游戏让我玩上瘾了！"
  • "你的演讲很真诚，我感受到了你的热情。"
  • "你的项目真的能帮助到很多人。"

结束时： 邀请家长扫码查看孩子的 EdgeOne 个人网站和灵光作品集，网站开着 EdgeOne 加速，打开飞快，家长现场就能转发到家族群，这是最自然的分发和转化触点。

家长视角：看见一个未来创新者

一天体验课做下来，家长能清晰看见：

| 家长关心的 | 课程中体现为 |

|-----------|------------|

| 孩子关心什么社会问题？ | 角色选择 → 项目方向 → 直接反映内心关注的社会议题 |

| 孩子有没有解决问题的能力？ | 发现问题 → 设计方案 → AI 原型 → 完整创新闭环 |

| 孩子是否具备用户思维？ | "谁需要这个方案？"——从理解人开始的创新训练 |

| 孩子的表达能力怎样？ | 戏剧表演 + 路演展示 → 讲清楚"我为什么想改变这件事" |

| 孩子与人合作如何？ | 小组分工 + 互评环节 → 社交与协作模式 |

| 孩子遇到困难怎么办？ | 灵光创作调试过程 → 抗挫力 + 迭代思维 |

| 孩子能创造什么？ | 海报 + 游戏 + 作品集 + 个人网站 → 从想法到产品的完整输出 |

| 孩子适合继续深入吗？ | 全程参与度 + 社会关怀持续度 → 是否继续培养创新素养 |

家长的卡点通常在于：

  • ❓ "我的孩子能学会吗？" → 一天做出作品 + 个人网站就是答案
  • ❓ "手机上课会不会只是玩？" → 有结构的创新流程，从问题出发到方案落地，目的明确
  • ❓ "学 AI 有用吗？" → 让孩子体验"用技术解决真实问题"，创新思维比 AI 技能更持久

常见问题（FAQ）

Q：为什么同时用豆包和灵光？会不会太复杂？

A：分工明确——豆包做对话和生图，灵光做游戏和作品集。孩子只需要记住"豆包用来想和画，灵光用来做和展"，很自然。

Q：全程用手机，效果会不会不如电脑？

A：手机是孩子最熟悉的设备，降低技术门槛，让孩子把注意力放在发现问题和设计方案上。灵光的移动端体验本身就很完善。

Q：只有一天，孩子能做出什么？

A：一个完整的创新闭环：发现社会问题 → 设计方案 → AI 海报 → AI 互动游戏 → 作品集 → 个人网站 → 路演展示。每个孩子会带着自己的社会创新方案离开。

Q：家长可以旁听吗？

A：建议下午路演环节开放家长参与。结束时家长扫码就能看到孩子的作品集和个人网站，是最自然的成果展示。

Q：跟两天课有什么区别？

A：一天体验课是"启蒙"，让孩子体验"用 AI 解决一个问题"的完整过程。两天课是"深耕"，增加更深入的社会调研、方案打磨和路演训练。

Q：戏剧教育跟 AI 有什么关系？

A：戏剧让孩子先"打开自己"，才谈得上"发现真实的问题"。没有对人和社会的感知，AI 创作就缺乏灵魂。

EdgeOne 部署速查（教练课前准备）

课前配置清单

EdgeOne 部分由教练/助教在电脑端操作，孩子只需把 HTML 代码发过来即可。
  • [ ] 注册腾讯云账号并完成实名认证
  • [ ] 创建 COS 存储桶，开启静态网站托管（设置 → 静态网站 → 开启）
  • [ ] 在 EdgeOne 控制台添加域名，源站类型选择 COS 源，绑定刚创建的存储桶
  • [ ] 安装 C polar（腾讯云 COS 客户端）在教练电脑上，配置好密钥登录
  • [ ] 准备 3-5 个通用 HTML 模板（豆包提前生成好，应对时间不够的情况）

课前配置步骤


1. COS 存储桶创建
   → 腾讯云控制台 → 对象存储 COS → 创建存储桶
   → 名称：kids-project-xxx（自定义）
   → 地域：选择离上课地点最近的
   → 开启"静态网站" → 默认首页 index.html

2. EdgeOne 加速配置
   → 腾讯云控制台 → EdgeOne → 站点列表 → 添加站点
   → 源站配置：选择 COS 源 → 选择刚创建的存储桶
   → HTTPS 证书：自动配置（EdgeOne 自带免费证书）

3. C polar 客户端安装
   → 官网下载 C polar（支持 Mac / Windows）
   → 用腾讯云 API 密钥登录
   → 在左侧看到刚创建的 COS 存储桶

现场部署操作


已安装 C polar 的情况下：
1. 收到孩子发送的 HTML 代码
2. 粘贴保存为 index.html（文件名固定）
3. 打开 C polar → 选择存储桶
4. 右键点击存储桶 → 上传文件 → 选择 index.html
5. 上传完成后右键文件 → "复制链接"
6. 把链接发回孩子微信
# 链接格式如：https://xxx.cos.ap-guangzhou.myqcloud.com/index.html
# 通过 EdgeOne 加速后可换成自定义域名

时间不够时的简化方案


助教提前用豆包生成 3-5 个主题模板 HTML：
  template-ocean.html（海洋主题）
  template-school.html（校园主题）
  template-space.html（科技主题）
  template-forest.html（自然主题）
  template-generic.html（通用主题）

现场只需：
1. 找到对应主题的模板
2. 用 VS Code 或记事本替换名字和链接
3. 直接上传 COS
4. 耗时：每人 2-3 分钟
