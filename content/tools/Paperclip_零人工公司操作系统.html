什么是 Paperclip？

Paperclip 是一个开源 AI Agent 编排平台，定位为"零人工公司"的操作系统——让你招募、管理和编排 AI 智能体团队，在最小人工干预下运营一家"企业"。

快速安装

多种部署方式，从本地到云端，三选一即可。

⚡

有 Node.js 环境即可
npx paperclipai onboard --yes

有 Docker 环境即可

创建你的第一家 AI 公司

Step 1
注册管理员
浏览器打开 localhost:3100，注册管理员账号，进入仪表盘
Step 2
创建公司 + 使命陈述
填写公司名称和使命陈述——这是所有 Agent 决策的顶层依据
Step 3
招聘 CEO Agent
选择适配器（推荐 Claude Code），设置心跳间隔和月预算。CEO 启动后会主动"招聘"下属
Step 4
设定目标
在公司目标中添加具体任务，指派给 CEO。CEO 自动分解给下属 Agent 执行并汇报
好的使命示例
"开发一款 AI 笔记应用，做到 100 万美元 MRR"
不好的使命示例
"做一些 AI 相关的东西"

• 审计日志 — 审查每个 Agent 的历史决策

心跳调度 Heartbeats

Agent 不会一直运行，而是按计划"醒来 → 检查任务 → 执行 → 报告 → 睡觉"。这是 Paperclip 的核心调度机制。

睡眠 → [心跳] → 醒来 → 检查任务队列 → 有任务则执行/汇报 · 无任务则直接睡觉
Agent 角色推荐间隔理由
CEO / 高管3600 秒（1小时）战略层不需要频繁响应
编程 Agent600 秒（10分钟）代码任务需要较频繁检查
内容 Agent86400 秒（1天）内容创作按天排期
客服 Agent300-600 秒客户等待有心理阈值
最低安全间隔30 秒低于此值可能失控
⏰ 定时心跳
Agent 按 cron 计划自动唤醒，完成例行任务后自动休眠
⚡ 事件驱动
GitHub push、新订单、监控告警等事件触发 Agent 即时响应

预算控制 Budget Control

AI Agent 调用 API 会产生费用。预算控制不是限制，而是安全护栏——设好上限，让 Agent 在范围内自由发挥。

0-80%
正常运行
Agent 自主工作，无需干预
80%
预警通知
发出预警，人为评估是否追加
100%
硬停止
Agent 自动暂停工作
Agent 角色推荐适配器月预算建议
CEOClaude Sonnet$50-100/月
工程师Claude Sonnet$50-100/月
QAClaude Haiku$20-50/月
内容 AgentClaude Haiku$20-50/月

⚠️ 第一次启动时必须设置预算，否则 Agent 可能无限制调用 API

Skills 技能系统

Skills 是 Markdown 文件，教会 Agent 执行特定工作流。一次编写，所有 Agent 复用，无需重新训练模型。

目录结构
skills/ ├── code-review.md ├── deploy-to-vercel.md ├── write-blog-post.md ├── handle-customer-ticket.md └── ...
示例：deploy-to-vercel
# deploy-to-vercel ## 前置条件 - Vercel CLI 已安装 ## 步骤 1. vercel --prod 2. 确认部署 URL 可访问 3. 在 GitHub Release 标记版本 4. 通知相关 Agent ## 注意事项 - 失败时自动回滚

✅ 优势

⚠️ 不足

• 不适合受监管行业
• Agent 会自信地产生幻觉
• 大规模并行（2048+）尚未验证
• 当前偏技术用户取向

最佳 实践

1️⃣ 从小开始
先用 3-5 个 Agent（1 个协调者 + 2-4 个执行者），熟悉后再扩展到 10+
2️⃣ 写好使命陈述
使命是一切决策的源头。包含：做什么 + 为什么 + 衡量标准
3️⃣ 立即设置预算
起手 $20-50/月/Agent，从日志了解用量后再调整，设好预警通知
4️⃣ 合理分配模型
Haiku/轻量 → 路由、格式化、QA · Sonnet/重量 → 编程、策略、创造
5️⃣ 经常检查日志
前 48 小时是黄金调试期，大部分配置问题会在初期暴露
6️⃣ 文件驱动协作
Agent 之间通过共享文件通信，非直接对话。所有"沟通"可追溯可复盘
