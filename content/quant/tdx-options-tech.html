通达信 + miniQMT 期权期货量化交易 技术方案

基于通达信技术指标信号触发，通过 miniQMT 极速执行通道实现期权期货自动化交易

版本 v2.1 日期 2026-05-15 品类 股指期权 / 商品期货 / 商品期权 执行平台 miniQMT (xtquant)

1. 背景与目标

1.1 问题域

在期权期货交易中，交易者面临以下核心痛点：

  • 通达信提供强大的技术指标和公式系统，但无实盘交易接口，信号无法自动执行
  • 期权期货盯盘强度高、决策时效短，人工执行难以保证纪律性和响应速度
  • 多品种、多合约的跨市场套利需要程序化协调，手工无法胜任

1.2 方案目标

  • 信号自动化 — 通达信技术指标信号实时捕获，零人工干预完成交易执行
  • 低延迟 — 全链路端到端延迟控制在 500ms 以内（含网络传输）
  • 全品种覆盖 — 股指期权、商品期货、商品期权统一接入
  • 工程化风控 — 资金管理、仓位控制、异常熔断三防体系
  • 可观测 — 全链路日志、实时监控、操作审计

1.3 适用读者

  • 熟悉通达信指标公式、希望将其转化为自动化策略的交易者
  • 具备 Python 基础、希望搭建个人量化交易系统的开发者
  • 需要为团队搭建期权期货交易基础设施的 IT 人员

3.2 延迟预算分解

端到端延迟目标：< 500ms（从通达信信号触发到委托到达柜台）

环节预算说明
T1: 通达信写入文件< 50ms条件预警触发到文件落盘
T2: 文件变更检测< 50mswatchdog 事件通知延迟
T3: 信号解析 + 风控< 50msJSON 解析 + 内存校验
T4: passorder 调用< 5ms本地进程间调用
T5: 柜台到交易所~200ms网络传输 + 交易所处理
合计< 355ms不含交易所排队时间
实测参考：沪市全链路约 25ms，深市约 3ms（2025年行业测试数据）。以上为保守设计预算。

3.3 数据一致性保障

文件锁机制

  • 通达信写入端：写文件前创建 .lock 哨兵文件，写完删除
  • 监控端：检测到 .lock 存在时跳过该文件，等待解锁
  • 超时：锁持有超过 5s 触发告警，强制解锁

去重保证

  • 每条信号包含唯一 signal_id（通达信时间戳+序号）
  • 引擎维护已处理队列（内存 + 持久化），去重后执行
  • 重启后加载历史 ID 到布隆过滤器，避免重复执行

乱序处理

  • 信号文件按时间戳递增写入，监控端严格按时间序处理
  • 发现时间戳回退时触发告警，暂停交易直到人工确认

幂等性

  • 同一信号 ID 的委托最多发送一次
  • 委托发送前检查订单状态（已报/已撤），避免重复下单

6.1 核心代码框架

# quant_engine.py — 监控引擎核心
import json
import os
import time
import logging
from pathlib import Path
from threading import Thread, Lock
from watchdog.observers import Observer
from watchdog.events import FileSystemEventHandler
from xtquant import xtdata, xttrader

class SignalHandler(FileSystemEventHandler):
    """通达信信号文件监控处理器"""

    def __init__(self, signal_path: str, engine):
        self.signal_path = Path(signal_path)
        self.engine = engine
        self.processed_ids = set()  # 去重集合
        self._lock = Lock()
        self._load_processed_ids()

    def on_modified(self, event):
        if not event.src_path.endswith('.jsonl'):
            return
        # 检测锁文件
        lock_file = self.signal_path.with_suffix('.lock')
        if lock_file.exists():
            # 通达信仍在写入，跳过本轮
            return
        self._process_file(event.src_path)

    def _process_file(self, filepath: str):
        """读取并处理信号文件"""
        with open(filepath, 'r', encoding='utf-8') as f:
            for line in f:
                line = line.strip()
                if not line:
                    continue
                try:
                    signal = json.loads(line)
                    self._handle_signal(signal)
                except json.JSONDecodeError as e:
                    logging.error(f"JSON解析失败: {e}, line: {line}")

    def _handle_signal(self, signal: dict):
        """处理单条信号：去重→风控→执行"""
        signal_id = signal.get('signal_id')
        if not signal_id or signal_id in self.processed_ids:
            return
        with self._lock:
            # 风控前置校验
            risk_result = self.engine.risk_controller.check(signal)
            if not risk_result.passed:
                logging.warning(f"风控拦截: {signal_id}, 原因: {risk_result.reason}")
                return
            # 执行交易
            self.engine.executor.execute(signal)
            self.processed_ids.add(signal_id)

    def _load_processed_ids(self):
        """重启时加载已处理ID，避免重复"""
        record_file = Path('data/processed_ids.parquet')
        if record_file.exists():
            # 加载历史 ID 到布隆过滤器
            pass

6.2 执行器实现

class OrderExecutor:
    """交易执行器 - 封装 xttrader.passorder()"""

    def __init__(self, account_id: str):
        self.account_id = account_id
        self.session = xttrader.TradeSession()
        self.session.connect()

    def execute(self, signal: dict) -> dict:
        """将信号转换为委托并发送"""
        direction = signal['direction']
        order_mode = self._direction_to_mode(direction)

        # 获取实时价格（如果 signal 中 price 为 0，则用市价单）
        price = signal.get('price', 0)
        if price == 0:
            price = self._get_market_price(signal['symbol'])

        # 调用 passorder
        order_id = xttrader.passorder(
            order_type=23,              # 限价单
            order_mode=order_mode,
            account_id=self.account_id,
            code=signal['symbol'],
            price=price,
            qty=signal['qty'],
            strategy_name="TDX_QUANT",
            order_remark=signal.get('signal_id', '')
        )

        logging.info(f"委托已发送: signal_id={signal['signal_id']}, order_id={order_id}")
        return {'order_id': order_id, 'status': 'sent'}

    def _direction_to_mode(self, direction: str) -> int:
        """将通达信方向转换为 xtquant order_mode"""
        mapping = {
            'buy_open': 0,    # 买入开仓
            'sell_close': 1,  # 卖出平仓
            'sell_open': 2,   # 卖出开仓
            'buy_close': 3,   # 买入平仓
        }
        mode = mapping.get(direction)
        if mode is None:
            raise ValueError(f"未知方向: {direction}")
        return mode

    def _get_market_price(self, symbol: str) -> float:
        # 获取最新行情价格
        tick = xtdata.get_full_tick([symbol])
        return tick[symbol]['lastPrice']

7. 策略框架

7.1 在通达信中编写指标公式

以下为示例：一个基于 MACD 金叉 + 成交量确认的股指期权买入开仓信号：

{通达信公式 - MACD金叉+量能确认}
// 适用: 沪深300股指期权 IO 合约

DIFF := EMA(CLOSE,12) - EMA(CLOSE,26);
DEA  := EMA(DIFF,9);
MACD := 2*(DIFF-DEA);

// 金叉条件: DIFF 上穿 DEA
CROSS_DIFF_DEA := CROSS(DIFF, DEA);

// 成交量确认: 当前成交量 > 5日均量的1.5倍
VOL_CONFIRM := VOL > MA(VOL,5) * 1.5;

// 信号输出
BUY_SIGNAL := CROSS_DIFF_DEA AND VOL_CONFIRM;

// 将信号输出到预警（在条件预警设置中绑定此公式）
DRAWTEXT(BUY_SIGNAL, LOW, 'BUY_IO');
注意：通达信公式中无法直接获取期权隐含波动率、Greeks 等数据。如需基于 Greeks 的期权策略（如 Delta 对冲、波动率交易），需要将数据源切换到 QMT，在 Python 端完成策略逻辑，通达信仅做辅助参考。

7.3 Python 侧策略编排

class StrategyRouter:
    """策略路由：根据信号来源分派到具体策略"""

    def __init__(self):
        self.strategies = {
            'trend_breakout': TrendBreakoutStrategy(),
            'option_buyer': OptionBuyerStrategy(),
            'basis_arbitrage': BasisArbitrageStrategy(),
        }

    def route(self, signal: dict) -> Optional[OrderSignal]:
        """路由决策：返回 OrderSignal 或 None（不执行）"""
        symbol = signal['symbol']
        strategy_type = self._classify(symbol)

        strategy = self.strategies.get(strategy_type)
        if not strategy:
            logging.warning(f"无匹配策略: {symbol}")
            return None

        return strategy.process(signal)

    def _classify(self, symbol: str) -> str:
        """根据合约代码判断策略类型"""
        if symbol.startswith('IO') or symbol.startswith('HO'):
            return 'option_buyer'
        if symbol.endswith('-C-') or symbol.endswith('-P-'):
            return 'option_buyer'
        if any(f in symbol for f in ['IF', 'IC', 'IH', 'RB', 'MA']):
            return 'trend_breakout'
        return 'trend_breakout'

8.1 风控架构

风控分为 三层防线，层层拦截：

8.3 风控控制器实现

class RiskController:
    """风控前置校验引擎"""

    def __init__(self, config: dict):
        self.config = config
        self.daily_pnl = 0.0           # 当日盈亏
        self.order_count_minute = 0   # 本分钟委托计数
        self.last_order_ts = 0        # 上次委托时间戳
        self.circuit_broken = False    # 熔断状态
        self.circuit_break_ts = 0     # 熔断触发时间

    def check(self, signal: dict) -> RiskResult:
        """风控前置校验，返回校验结果"""
        now = time.time()

        # 1. 熔断检查
        if self.circuit_broken:
            remaining = self.circuit_break_ts + \
                self.config['circuit_breaker_cooldown_min'] * 60 - now
            if remaining > 0:
                return RiskResult(False, f"熔断中，剩余 {int(remaining)}s")
            self.circuit_broken = False

        # 2. 白名单检查
        if signal['symbol'] not in self.config['whitelist_symbols']:
            return RiskResult(False, "合约不在白名单中")

        # 3. 频率检查
        if now - self.last_order_ts < float(
                self.config['min_order_interval_ms']) / 1000:
            return RiskResult(False, "下单间隔过短")

        # 4. 价格偏离检查
        current_price = self._get_current_price(signal['symbol'])
        deviation = abs(signal['price'] - current_price) / \
            (current_price or 1)
        if deviation > self.config['price_deviation_pct']:
            return RiskResult(False,
                f"价格偏离 {deviation:.2%} 超限")

        # 5. 日内亏损熔断
        if self.daily_pnl < float(self.config['daily_loss_limit_pct']) * \
                self._get_total_asset():
            self.circuit_broken = True
            self.circuit_break_ts = now
            return RiskResult(False, "日内亏损熔断触发")

        return RiskResult(True, "")

12.2 看门狗实现

# watchdog.py — 独立的进程看门狗，由 Windows Task Scheduler 拉起
import psutil
import subprocess
import time

class ProcessWatchdog:
    """监控关键进程，崩溃时自动重启"""

    PROCESSES = {
        'tdx': {'name': 'TdxW.exe', 'path': r'D:\quant\tdx\TdxW.exe'},
        'engine': {'name': 'python.exe', 'path': r'D:\quant\engine\main.py',
                    'args': ['--config', 'config.yaml']},
    }

    def check(self):
        for name, proc in self.PROCESSES.items():
            running = any(p.info['name'] == proc['name']
                         for p in psutil.process_iter(['name']))
            if not running:
                logging.warning(f"进程 {name} 未运行，尝试重启...")
                self._restart(name, proc)

    def _restart(self, name, proc):
        if name == 'engine':
            subprocess.Popen(['python', proc['path']] + proc.get('args', []))
        elif name == 'tdx':
            subprocess.Popen([proc['path']])
