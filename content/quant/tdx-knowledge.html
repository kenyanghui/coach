第一篇 · 通达信入门手册

从零开始学习通达信 —— 安装、界面、操作、分析、交易

1.1 通达信是什么

通达信是国内最主流的股票行情与交易软件之一，由财富趋势科技公司（科创板 688318.SH）开发。绝大多数券商都提供通达信定制版，覆盖沪深港美及期货、期权等多市场。

  • 行情 — 实时沪深 Level-2 行情、港股、美股、期货、外汇
  • 分析 — K线、分时图、技术指标（MACD、KDJ、RSI 等上百种）
  • 选股 — 条件选股、技术指标选股、基本面选股
  • 交易 — 内置交易模块，支持多券商
  • 扩展 — 支持公式编写、DLL 插件、自定义版面
一句话：通达信 = 看盘 + 分析 + 选股 + 交易，一站搞定。

1.7 K 线图

K 线（蜡烛图）包含四个价格：开盘价、收盘价、最高价、最低价。

  • 阳线（红色） — 收盘价 > 开盘价，上涨
  • 阴线（绿色/蓝色） — 收盘价 < 开盘价，下跌
  • 上影线 — 最高价到实体的部分
  • 下影线 — 最低价到实体的部分

分析周期

按 F8 切换周期：1分钟 → 5分钟 → 15分 → 30分 → 60分 → 日线 → 周线 → 月线 → 季线 → 年线

多图组合

  • Ctrl+2 — K线 + 成交量
  • Ctrl+3 — K线 + 成交量 + MACD
  • Ctrl+4 — K线 + 成交量 + MACD + KDJ

2.3 为什么行业普遍选用通达信

一句话

券商选择通达信，不是因为它最好，而是因为换掉它的成本太高。30年路径依赖+政策刚需+信创合规+行业标准，四重锁定让通达信的 B 端地位难以撼动。

原因一：30年先发优势，深度绑定券商基础设施

通达信成立于 1995 年，是国内最早的证券软件企业。经过近 30 年迭代，系统与券商柜台、结算系统深度耦合。迁移成本远高于使用成本。

原因二：B2B 模式 + 高粘性维护收入

维护服务占收入 41%，毛利率高达 93.7%。券商每年付维护费，形成长期绑定。

原因三：政策驱动的刚性升级

注册制改革、北交所设立、国密改造——每次改革通达信都是最先完成适配的厂商。

原因四：信创（国产替代）先发优势

率先完成统信/麒麟国产系统适配，发布鸿蒙 PC 版，是券商信创验收的首选。

原因五：技术指标的行业标准

通达信公式语言是事实上的行业标准，大量教程、书籍、策略源码都以通达信公式编写。

3.1 量化平台概览

目前国内主流券商量化交易平台是 迅投 QMT 和 恒生 PTrade，而通达信在量化生态中的角色是行情分析与信号生成：

恒生 PTrade恒生电子✅ 直接实盘偏股票/两融云端运行
通达信财富趋势❌ 仅分析全品种行情本地运行
聚宽/米筐第三方❌ 已停止实盘研究回测云端
趋势：实盘交易接口正在从第三方平台向券商自有平台集中。聚宽 2025 年已停止实盘接口。

miniQMT

QMT 的极简模式，将策略编写和实盘分离。外部 Python 通过 xtquant SDK 调用下单接口，比标准版更灵活。这也是通达信+量化方案的核心组件。

3.3 通达信在量化中的角色

通达信不是量化交易执行平台，但它扮演关键的信号生成角色：

通达信（技术指标公式 → 条件预警 → 输出信号） │ 写入 JSON Lines 信号文件 ▼ Python 监控引擎（watchdog 检测 → 解析 → 风控 → 决策） │ xttrader.passorder() ▼ miniQMT → 券商极速柜台 → 交易所

通达信的优势是指标公式系统业内最强，适合先做技术分析、条件选股生成信号，再通过 miniQMT 执行。2026 年新版内置了 TQ 策略功能，但尚在测试阶段。

3.6 关键实现

信号文件格式（JSON Lines）

# D:\signals\signals.jsonl
{"signal_id":"20260515_143021_001","symbol":"IO2506-C-4000","direction":"buy_open","price":125.0,"qty":1}

核心监控引擎（伪代码）

# quant_engine.py — 监控引擎核心
class SignalHandler(FileSystemEventHandler):
    def on_modified(self, event):
        if not event.src_path.endswith('.jsonl'): return
        self._process_file(event.src_path)

    def _process_file(self, filepath):
        with open(filepath, 'r') as f:
            for line in f:
                signal = json.loads(line)
                if self._dedup(signal): continue      # 去重
                if not self._risk_check(signal): continue  # 风控
                self._execute(signal)                 # 执行

    def _execute(self, signal):
        order_id = xttrader.passorder(
            order_type=23,                    # 限价单
            order_mode=mode_map[direction],   # 开/平方向
            account_id=ACCOUNT,
            code=signal['symbol'],
            price=signal['price'],
            qty=signal['qty'],
        )

3.7 风控体系

三层防线

第一层：前置校验（Python端） 资金充足性 → 仓位上限 → 下单频率 → 合约白名单 → 价格偏离度 第二层：委托拦截（miniQMT侧） 反向对冲自检 → 日内亏损熔断 → 连续撤单率监控 第三层：柜台风控（券商侧） 资金冻结 → 持仓检查 → 交易所熔断

风控参数示例

资金类：

  • 单笔最大委托金额：10 万
  • 总仓位上限：总资产 70%
  • 单一品种上限：30%

熔断类：

  • 日内亏损 < -5% 停盘
  • 熔断冷却时间：30 分钟
  • 价格偏离现价 > 3% 拦截
重要：风控是量化交易的生命线。回测再好的策略，没有风控也会在一次黑天鹅中归零。所有参数需根据实际资金量和风险承受能力调整。

    def route(self, signal):
        symbol = signal['symbol']
        if symbol.startswith('IO'):        # 股指期权
            return OptionBuyerStrategy().process(signal)
        elif symbol.startswith('IF'):      # 股指期货
            return TrendBreakoutStrategy().process(signal)
        elif symbol.startswith('MA'):      # 商品期货
            return BasisArbitrageStrategy().process(signal)
        return None

3.9 竞争差异分析

核心观点

通达信+QMT 这套体系不是为了"打赢机构"，而是为了"不和机构在同一战场"。找到机构因为规模、成本、合规、速度等原因无法进入的缝隙市场，在这些市场里用灵活性建立护城河。

机构不擅长做的事

  • 小容量策略（容量 < 5000 万，养不活团队）
  • 低流动性品种（大资金进出成本高）

  • 冷门期权合约（市场深度不够）
  • 政策驱动交易（机构合规限制多）

3.11 最终建议

资金 < 500 万

这套方案完全够用。专注于跨期套利、期权价差等"小而美"策略，在机构不关注的领域建立优势。预期年化 15~30%。

资金 500 万~5000 万

仍可用但需逐步升级：直接对接 CTP/LDP 柜台，引入更多数据源和复杂策略。

资金 > 5000 万

通达信+QMT 的天花板到了。需建立自己的交易团队和基础设施，或以机构身份使用专业交易系统。

期权期货为主

这套方案可以比股票量化更充分地发挥优势。期权期货市场流动性分层明显，天然适合小资金寻找定价偏差。

最终结论

通达信+QMT 的正确用法不是在机构的地盘上和他们竞争，而是找到机构因为规模、成本、合规、速度等原因无法进入的缝隙市场，在这些市场里用灵活性建立护城河。

合理预期：年化 15~30%，最大回撤 < 10%。如果能做到这一点，你已经跑赢了 90% 的投资者和大部分量化机构的产品。

起步路径

第一步：验证链路（1~2周）

最简单趋势跟踪策略跑通全套链路（通达信→QMT→实盘），最小资金验证系统稳定性。

第二步：回测体系（持续迭代）

通达信历史数据导出，Python 建立回测框架，检验极端行情表现。

第三步：找到利基（3~6个月）

在对特定品种的深度理解中发现"机构进不来而你能做"的机会。

第四步：组合运行（6~12个月）

2~3 个不相关的利基策略组合运行，降低回撤，持续增长。
